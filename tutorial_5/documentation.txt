2.1. Why are the variables declared as private?
Variables are declared as private to achieve encapsulation. This restricts direct access to the variables from outside the Student class, ensuring the data is hidden, secure, and cannot be changed directly by other parts of the program.

2.2. What happens if the variables are declared public?
If the variables are declared public, any other class can directly access and modify them. This breaks encapsulation and allows for uncontrolled or potentially invalid changes to the student's data, compromising the security of the information system.

2.3. Why do we use getters and setters?
Getters and setters provide controlled access to private variables. Getters allow outside classes to safely read the variable's value, while setters allow them to modify the value. This setup allows the class to include validation rules inside the setters before the data is actually changed.
